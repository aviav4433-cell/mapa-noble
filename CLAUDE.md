# MAPA-NOBLE ERP — הקשר קבוע לסשן עבודה

קרא קובץ זה במלואו לפני כל שינוי. הוא חוסך שעות של הבנת המערכת מחדש.

## מה המערכת

מערכת ERP לשני עסקים: MAPA (השכרת מפות וטקסטיל לאירועים) ו-NOBLE (מכבסה תעשייתית).
הבעלים: אבי. אינו מתכנת — כל הנחיה אליו חייבת להיות פשוטה, צעד-אחר-צעד, בעברית.

## ארכיטקטורה

- **שרת:** Google Apps Script, קובץ יחיד `Code.gs` (~1,300 שורות). נקודת כניסה יחידה `doPost`.
- **נתונים:** Google Sheets — 31 טבלאות. הגדרת העמודות באובייקט `TABLES` בראש Code.gs.
- **ממשק:** `index.html` יחיד (~2,650 שורות), עברית RTL, מתארח ב-GitHub Pages.
- **ריפו:** `aviav4433-cell/mapa-noble`, ענף `main`. האתר: https://aviav4433-cell.github.io/mapa-noble/
- **Endpoint:** מוגדר במשתנה `API_URL` בראש הסקריפט של index.html. אסור לשנות אותו.

## פרוטוקול תקשורת ממשק↔שרת (קריטי — כאן כבר נולדו באגים)

- הממשק שולח `POST` עם גוף JSON: `{ action, payload, token?, password?, user? }`
- **הטוקן חייב להיות ברמה העליונה של הגוף** (`body.token`), והשרת מקבל אותו גם מ-`payload.token` (תאימות לאחור). אל תשבור את זה.
- כניסה: action `login` עם `{ username, password }` ב-payload. השרת מחזיר `token` (סשן 6 שעות ב-CacheService).
- "זכור אותי": `rememberLogin` מנפיק `remember_token` (נשמר hashed ב-ScriptProperties), `loginByRemember` מחדש סשן.
- הסשן ב-Cache חייב לכלול `{ employee_id, role, name }` — השדה `name` נחוץ לבדיקות ההרשאה ב-`route`.
- **טקסטים ארוכים (אצווה G1):** `getAll` מחזיר שדות טקסט ארוכים (notes.text, tasks.details, meetings.protocol, task_comments.text) קצוצים ל-300 תווים עם דגל `<שדה>_trunc: 1`. הנתונים בגיליון לא נחתכים לעולם. שליפה מלאה: פעולת `getFullTexts {table, ids}` (קריאה בלבד). בממשק: `ensureFullTexts` לפני כל הצגה/עריכה של רשומה קטועה. **אסור לפתוח טופס עריכה של שדה כזה בלי לשלוף קודם את הטקסט המלא — אחרת שמירה תדרוס את המקור.**

## אבטחה — חוקים שאסור להפר

1. סיסמאות עובדים נשמרות רק כ-hash: `sha256WithSalt(password, employee_id)` = `sha256Hex(salt + ':' + password)`. לעולם לא טקסט גלוי.
2. המשתנה `PASSWORD` בראש Code.gs הוא שריד היסטורי שהוחלף למחרוזת אקראית. אל תחזיר אותו לערך קריא ואל תבנה עליו פיצ'רים.
3. אל תוסיף פונקציות עזר שמכילות סיסמאות בטקסט גלוי (כמו initAdminPassword שהוסרה).
4. אל תכתוב סיסמאות או טוקנים לקבצים בריפו, כולל לקובץ זה.
5. פריסת ה-Web App: הרצה בתור "אני", גישה "כל אחד". האבטחה היא במנגנון הטוקנים, לא במסך של גוגל.

## פריסה — התהליך הנכון

- שינוי ב-index.html → commit ל-main → GitHub Pages מתעדכן תוך דקות. רענון קשיח (Ctrl+Shift+R) אצל המשתמש.
- שינוי ב-Code.gs → אבי מדביק את הקובץ המלא בעורך Apps Script → שמירה → פריסה ← ניהול פריסות ← **עריכת הפריסה הקיימת** ← גרסה חדשה ← פרוס. **לעולם לא פריסה חדשה** — זה משנה את הכתובת ושובר את הממשק.
- כשמכינים ל-אבי קובץ שרת: תמיד קובץ שלם להחלפה מלאה (Ctrl+A והדבק), לא קטעים להדבקה ידנית.

## שיטת עבודה בריבוי משימות

1. פתח טבלת מעקב: מספר משימה, תיאור, סטטוס (ממתין/בוצע/נבדק).
2. בצע בקבוצות של עד 10. אחרי כל קבוצה: בדיקת תחביר לשני הקבצים (node --check או new Function).
3. שינוי שנוגע גם בשרת וגם בממשק — בצע את שני הצדדים באותה קבוצה, לעולם לא בנפרד.
4. בסוף: דו"ח סטטוס מלא + רשימת בדיקות ידניות קצרה לאבי (מה ללחוץ, מה אמור לקרות).

## מלכודות ידועות

- קבצים שאבי מעלה לצ'אט עלולים להיקטע — ודא שהקובץ מסתיים ב-`</html>` (ממשק) או בפונקציה שלמה (שרת). אם קטוע, משוך מהריפו: `raw.githubusercontent.com/aviav4433-cell/mapa-noble/main/index.html`
- אל תסמוך על מסמכי העברה מסשנים קודמים לגבי מצב הקוד — קרא את הקוד עצמו. כבר קרה שמסמך תיאר גרסה ישנה.
- `route` בודק הרשאות לפי שם עובד (`user`), לא לפי id. פעולות מנהל דורשות עובד פעיל בתפקיד 'מנהל'.
- פעולות כתיבה עוברות LockService ו-audit; פעולות קריאה (ברשימת READ_ONLY_ACTIONS) לא. פעולה חדשה שכותבת לגיליון — אל תוסיף אותה ל-READ_ONLY_ACTIONS.
- עמודת `pin` בטבלת העובדים היא שריד מהכניסה הישנה ואינה בשימוש.
